### 创新点1：多智能体辩论反思机制（Multi-Agent Debate with Self-Reflection）

#### 核心思想
让多个机器人角色对应的 LLM 实例各自生成动作计划后进行"交叉辩论"，互相纠错，然后由"裁判 LLM"综合出最终计划；执行失败后自动触发反思链进行重规划。

#### 为什么出彩（评委视角）

1. **对抗幻觉（Hallucination）**：单 LLM 容易产生不合理动作（如穿过桌子、抓取超出臂展的物体），多 LLM 交叉验证可显著抑制幻觉。
2. **可解释性强**：辩论过程本身就是"决策日志"，报告中可直接展示。
3. **学术前沿**：2023-2024 年多篇顶会论文（如 Du et al. 2023 "Improving Factuality and Reasoning in Language Models through Multiagent Debate"）验证了多 agent 辩论的有效性。
4. **零成本增加**：仅需多次 API 调用，不需要任何模型修改。

#### 实现难度
⭐⭐⭐（3/5）—— 需要设计辩论流程和融合机制，但无复杂算法。

#### 成功率提升预期
+15%~25%（尤其在复杂多步骤任务中）

#### 详细实现方案

**架构设计：**

```
┌───────────────────────────────────────────────────┐
│                   Orchestrator                    │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐          │
│  │ Agent A │  │ Agent B │  │ Agent C │          │
│  │(Robot 1)│  │(Robot 2)│  │(Robot 3)│          │
│  └────┬────┘  └────┬────┘  └────┬────┘          │
│       │ 各自规划    │            │               │
│       ▼            ▼            ▼                │
│  ┌─────────────────────────────────┐             │
│  │        Debate Round (LLM)       │             │
│  │  Agent A critiques B & C       │             │
│  │  Agent B critiques A & C       │             │
│  │  Agent C critiques A & B       │             │
│  │         ↓  Refine              │             │
│  │  Judge LLM → Final Plan        │             │
│  └─────────────────────────────────┘             │
│       ↓                                          │
│   Execute → if Fail → Self-Reflection Loop       │
└───────────────────────────────────────────────────┘
```

**伪代码：**

```python
# --- dialog_prompter.py 核心逻辑 ---
import json
import ollama  # 或 openai 兼容接口

class DebatePlanner:
    def __init__(self, model="qwen2.5:7b", max_debate_rounds=2, max_retries=3):
        self.model = model
        self.max_debate_rounds = max_debate_rounds
        self.max_retries = max_retries

    def plan(self, observation: dict) -> dict:
        """
        主入口：通过多智能体辩论生成最终动作计划
        """
        # Step 1: 各 Agent 独立生成初始计划
        plans = {}
        for robot_id in observation["robots"]:
            plans[robot_id] = self._generate_initial_plan(robot_id, observation)

        # Step 2: 辩论轮次
        for round_idx in range(self.max_debate_rounds):
            critiques = {}
            for robot_id in observation["robots"]:
                other_ids = [r for r in observation["robots"] if r != robot_id]
                other_plans = {oid: plans[oid] for oid in other_ids}
                critiques[robot_id] = self._critique_others(
                    robot_id, plans[robot_id], other_plans, observation
                )

            # 根据批评修正计划
            for robot_id in observation["robots"]:
                plans[robot_id] = self._refine_plan(
                    robot_id, plans[robot_id], critiques, observation
                )

        # Step 3: Judge 综合出最终计划
        final_plan = self._judge_synthesize(plans, observation)

        return final_plan

    def _generate_initial_plan(self, robot_id: str, obs: dict) -> str:
        prompt = f"""You are controlling Robot [{robot_id}] in a MuJoCo environment.

OBSERVATION:
- Your position: {obs["robots"][robot_id]["position"]}
- Your gripper state: {obs["robots"][robot_id]["gripper"]}
- Objects in scene: {obs["objects"]}
- Goal: {obs["goal"]}
- Other robots: {[r for r in obs["robots"] if r != robot_id]} (plans pending)

Generate a step-by-step action plan for Robot [{robot_id}] ONLY.
Output format:
<PLAN>
Step 1: ...
Step 2: ...
</PLAN>
"""
        return self._call_llm(prompt)

    def _critique_others(self, my_id: str, my_plan: str,
                         other_plans: dict, obs: dict) -> str:
        others_text = "\n".join(
            f"[{rid}]'s plan:\n{plan}" for rid, plan in other_plans.items()
        )
        prompt = f"""You are Robot [{my_id}]. Review the plans of OTHER robots and critique them.

YOUR PLAN:
{my_plan}

OTHER ROBOTS' PLANS:
{others_text}

SCENE CONTEXT: {obs["goal"]}

Critique EACH other robot's plan on:
1. Will any action cause collision with you?
2. Are they trying to grab an object you need?
3. Is their plan physically feasible?
4. Is there a better coordination strategy?

Output format:
<CRITIQUE>
For Robot [X]: ...
For Robot [Y]: ...
</CRITIQUE>
"""
        return self._call_llm(prompt)

    def _refine_plan(self, robot_id: str, my_plan: str,
                     all_critiques: dict, obs: dict) -> str:
        critiques_text = "\n".join(
            f"[{rid}]'s critique:\n{c}" for rid, c in all_critiques.items() if rid != robot_id
        )
        prompt = f"""You are Robot [{robot_id}]. Improve your plan based on critiques from others.

YOUR ORIGINAL PLAN:
{my_plan}

CRITIQUES RECEIVED:
{critiques_text}

SCENE: {obs["goal"]}

Revise your plan addressing the valid critiques. Keep the output format:
<PLAN>
Step 1: ...
Step 2: ...
</PLAN>
"""
        return self._call_llm(prompt)

    def _judge_synthesize(self, plans: dict, obs: dict) -> dict:
        plans_text = "\n".join(
            f"[{rid}]:\n{plan}" for rid, plan in plans.items()
        )
        prompt = f"""You are the JUDGE. Synthesize all robot plans into ONE coherent action sequence.

ALL PLANS:
{plans_text}

GOAL: {obs["goal"]}

1. Resolve any conflicts between plans.
2. Determine the correct execution order.
3. Output the FINAL action commands for each robot.

Output format (strict JSON):
{{
    "actions": {{
        "robot_1": ["move_to(x,y,z)", "grasp(obj_id)", ...],
        "robot_2": ["move_to(x,y,z)", ...]
    }},
    "reasoning": "...",
    "execution_order": ["robot_1:step1", "robot_2:step1", ...]
}}
"""
        resp = self._call_llm(prompt)
        return json.loads(self._extract_json(resp))

    def execute_with_reflection(self, observation: dict, env, max_retries: int = 3):
        """
        执行 → 失败检测 → 反思重试
        """
        for attempt in range(max_retries):
            plan = self.plan(observation)

            # 执行动作
            result = env.step(plan["actions"])
            success = self._check_success(result, observation["goal"])

            if success:
                return {"status": "success", "plan": plan, "attempt": attempt + 1}

            # 反思：分析失败原因，修正 observation 描述
            failure_reason = self._reflect_on_failure(plan, result, observation)
            observation["failure_history"] = observation.get("failure_history", []) + [
                failure_reason
            ]

        return {"status": "failed", "reason": "Max retries exceeded"}

    def _reflect_on_failure(self, plan: dict, result: dict, obs: dict) -> str:
        prompt = f"""Your previous plan FAILED. Analyze WHY.

PREVIOUS PLAN: {plan}
RESULT: {result}
GOAL: {obs["goal"]}
PREVIOUS FAILURES: {obs.get("failure_history", [])}

Identify the ROOT CAUSE of failure:
1. Was the plan physically impossible?
2. Was there an unanticipated collision?
3. Was the object state misunderstood?
4. Were the robots' actions conflicting?

Output the ANALYSIS and a WARNING to guide the next attempt:
<REFLECTION>
ANALYSIS: ...
WARNING: ...
</REFLECTION>
"""
        return self._call_llm(prompt)

    def _call_llm(self, prompt: str) -> str:
        resp = ollama.generate(model=self.model, prompt=prompt)
        return resp["response"]

    def _extract_json(self, text: str) -> str:
        import re
        match = re.search(r'\{.*\}', text, re.DOTALL)
        return match.group(0) if match else "{}"

    def _check_success(self, result: dict, goal: str) -> bool:
        # 根据环境和目标判断成功条件
        # 例如：目标物体是否到达目标位置
        return result.get("task_complete", False)
```

**Prompt 设计要点：**
- 为每个阶段设计**独立的角色身份**（"你是 Robot X" vs "你是裁判"），切换身份时改变 system prompt。
- 要求输出**结构化标签**（`<PLAN>`, `<CRITIQUE>`, `<REFLECTION>`），便于解析。
- 在 Judge prompt 中明确指定 JSON 格式，确保可直接解析为动作命令。

### 创新点2：结构化思维链空间推理（Structured CoT Spatial Reasoning）

#### 核心思想
强制 LLM 按照固定的四阶段流程输出——**[OBSERVE] → [ANALYZE] → [PLAN] → [ACT]**，将空间推理显式化，避免"一步到位"的跳跃式错误决策。

#### 为什么出彩（评委视角）

1. **基于认知科学理论**：CoT（Chain of Thought）已被证明能显著提升 LLM 推理能力（Wei et al., 2022）。
2. **空间推理是具身智能的核心难点**：LLM 本身缺乏空间感知能力，结构化 CoT 强制其"停下来想一想"，弥补空间推理短板。
3. **与动作空间解耦**：四阶段输出中 [ACT] 是唯一需要解析成动作的部分，前三阶段提供了丰富的可解释性。

#### 实现难度
⭐⭐（2/5）—— 纯 Prompt 设计，只需调整输出格式要求。

#### 成功率提升预期
+10%~20%（在需要精确空间推理的任务中尤为明显）

#### 详细实现方案

**Prompt 设计：**

```
SYSTEM PROMPT (设置角色与输出格式):

You are a robot control system with spatial reasoning capabilities. 
You MUST follow this EXACT output structure for EVERY response:

========== FORMAT SPECIFICATION ==========

[OBSERVE]
- List all objects with their 3D positions (x, y, z)
- List all robots with their positions and gripper states
- Note any spatial constraints (walls, obstacles, reach limits)

[ANALYZE]
- Which robot is closest to each target object? (compute distances)
- Are there any potential collisions? (check paths)
- What dependencies exist between sub-tasks?
- What is the optimal execution order?

[PLAN]
Robot_1:
  Step 1: ...
  Step 2: ...
Robot_2:
  Step 1: ...
  Step 2: ...
(Specify target coordinates for each move)

[ACT]
{
  "actions": {
    "robot_1": [{"type": "move", "target": [x, y, z]}, ...],
    "robot_2": [{"type": "move", "target": [x, y, z]}, ...]
  }
}

========== END FORMAT ==========

CRITICAL RULES:
1. In [OBSERVE], use EXACT coordinates from the observation. Do NOT invent numbers.
2. In [ANALYZE], explicitly compute which robot is closer to each object.
3. In [PLAN], specify the target coordinates for every movement.
4. In [ACT], output ONLY valid JSON actions.
5. NEVER plan a movement to a coordinate that would cause collision.
```

**伪代码（解析与执行）：**

```python
# --- plan_prompter.py ---

SPATIAL_COT_PROMPT = """{system_prompt}

CURRENT OBSERVATION:
Robots:
{robots_state}

Objects:
{objects_state}

Goal: {goal}

Previous actions: {history}

Now, reason step by step following the [OBSERVE]→[ANALYZE]→[PLAN]→[ACT] format.
"""

class SpatialCoTPlanner:
    def __init__(self, model="qwen2.5:7b"):
        self.model = model
        self.system_prompt = SYSTEM_PROMPT  # 上面定义的

    def plan(self, obs: dict) -> dict:
        prompt = self._build_prompt(obs)
        raw_response = self._call_llm(prompt)

        # 解析四阶段输出
        sections = self._parse_sections(raw_response)
        observe = sections.get("OBSERVE", "")
        analyze = sections.get("ANALYZE", "")
        plan_text = sections.get("PLAN", "")
        actions = sections.get("ACT", "{}")

        # 验证空间一致性
        validated_actions = self._validate_spatial(actions_json=actions, obs=obs)

        return {
            "actions": validated_actions,
            "observe": observe,
            "analyze": analyze,
            "plan": plan_text,
        }

    def _parse_sections(self, text: str) -> dict:
        """解析 [OBSERVE] [ANALYZE] [PLAN] [ACT] 四个区块"""
        import re
        sections = {}
        patterns = ["OBSERVE", "ANALYZE", "PLAN", "ACT"]
        for i, tag in enumerate(patterns):
            pattern = rf'\[{tag}\]\s*(.*?)(?=\[{"|".join(patterns[i+1:])}\]|$)'
            match = re.search(pattern, text, re.DOTALL)
            if match:
                sections[tag] = match.group(1).strip()
        return sections

    def _validate_spatial(self, actions_json: str, obs: dict) -> dict:
        """
        空间验证：检查动作目标坐标是否合法
        - 是否在机器人臂展范围内？
        - 是否会穿过物体/墙壁？
        """
        actions = json.loads(actions_json)
        validated = {}
        for robot_id, action_list in actions.items():
            validated[robot_id] = []
            robot_pos = obs["robots"][robot_id]["position"]
            for action in action_list:
                if action["type"] == "move":
                    target = action["target"]
                    # 检查距离是否合理（机器人臂展通常 < 1.5m）
                    dist = self._euclidean(robot_pos, target)
                    if dist > 1.5:
                        # 超出臂展，生成中间移动点
                        midpoint = self._interpolate(robot_pos, target)
                        validated[robot_id].append(
                            {"type": "move", "target": midpoint}
                        )
                    validated[robot_id].append(action)
                    robot_pos = target  # 更新参考位置
                else:
                    validated[robot_id].append(action)
        return validated

    def _euclidean(self, a, b):
        return sum((ai - bi) ** 2 for ai, bi in zip(a, b)) ** 0.5
```

**增强版：可视化空间推理锚点（Anchor Points）**

在 [ANALYZE] 阶段要求 LLM 输出关键的"空间锚点"：

```
[ANALYZE]
Key Anchor Points:
  - object_A_position: (0.5, 0.0, 1.2)  ← robot_1 可从上方接近
  - destination_zone: (1.8, 0.0, 0.3)   ← robot_2 侧方接近更优
  - collision_risk: robot_1 路径经过 (1.0, 0.0, 0.8) 与 robot_2 路径交叉
  → 建议: robot_1 先到达 (1.0, 0.3, 0.8) 绕行，robot_2 等待 2 步
```

### 创新点3：动态角色分配与任务分解（Dynamic Role Assignment）

#### 核心思想
在每个决策周期，LLM 首先分析当前任务目标，将其分解为子任务，然后**动态地将子任务分配给最合适的机器人**（基于距离、空闲状态、能力），而非预先固定角色。

#### 为什么出彩（评委视角）

1. **自适应性强**：传统方法需要人为定义"机器人1负责抓取、机器人2负责放置"，而本方法根据实时状态动态调整。
2. **与多机器人协同的本质契合**：比赛中任务多变，"谁最适合做某件事"是动态变化的。
3. **高效并行**：合理的角色分配可以让多机器人并行工作，显著减少总任务时间。

#### 实现难度
⭐⭐（2/5）—— 核心是 Prompt 中增加角色分配逻辑，外加一个简单的距离计算辅助。

#### 成功率提升预期
+8%~15%（在需要并行操作的任务中）

#### 详细实现方案

**Prompt 设计：**

```
TASK DECOMPOSITION & ROLE ASSIGNMENT PROMPT:

You are a multi-robot task coordinator. Given a task goal and current state,
you must decompose the goal into sub-tasks and assign each sub-task to the
most suitable robot.

RULES:
1. One sub-task can be assigned to ONE robot at a time
2. Assign based on: distance to target, gripper state (free/occupied), current workload
3. Sub-tasks should be PARALLEL when possible
4. Some sub-tasks have dependencies (e.g. must grab before place)
5. ALL robots should stay busy unless no useful task exists

OUTPUT FORMAT:
<TASK_DECOMPOSITION>
Sub-task 1: [description] → assigned to Robot [X] (reason: ...)
Sub-task 2: [description] → assigned to Robot [Y] (reason: ...)
...
Dependencies: [Sub-task 2 depends on Sub-task 1]

<ROLE_ASSIGNMENT>
Robot X: [primary role] → Sub-tasks: [1, 3]
Robot Y: [support role] → Sub-tasks: [2]
</ROLE_ASSIGNMENT>

<ACTION_PLAN>
Robot X:
  Step 1: ...  # Sub-task 1
  Step 2: ...  # Sub-task 3
Robot Y:
  Step 1: ...  # Sub-task 2
</ACTION_PLAN>
```

```python
# --- plan_prompter.py ---

class DynamicRolePlanner:
    def __init__(self, model="qwen2.5:7b"):
        self.model = model

    def plan(self, obs: dict) -> dict:
        # Step 1: 硬编码的距离计算作为"锚点"，辅助 LLM
        distance_matrix = self._compute_distance_matrix(obs)

        # Step 2: 构建增强观测（含距离信息）
        enhanced_obs = self._enhance_observation(obs, distance_matrix)

        # Step 3: LLM 进行任务分解与角色分配
        prompt = self._build_role_assignment_prompt(enhanced_obs)
        response = self._call_llm(prompt)

        # Step 4: 解析输出
        result = self._parse_role_response(response)
        return result

    def _compute_distance_matrix(self, obs: dict) -> dict:
        """计算每个机器人到每个物体的距离"""
        matrix = {}
        for robot_id, robot_info in obs["robots"].items():
            r_pos = robot_info["position"]
            matrix[robot_id] = {}
            for obj_id, obj_info in obs["objects"].items():
                o_pos = obj_info["position"]
                dist = sum((r_pos[i] - o_pos[i])**2 for i in range(3)) ** 0.5
                matrix[robot_id][obj_id] = round(dist, 3)
        return matrix

    def _enhance_observation(self, obs: dict, dist_matrix: dict) -> str:
        robots_text = ""
        for robot_id, info in obs["robots"].items():
            robots_text += f"""
Robot [{robot_id}]:
  Position: {info['position']}
  Gripper: {info['gripper']}
  Distances to objects:
"""
            for obj_id, dist in dist_matrix[robot_id].items():
                robots_text += f"    → {obj_id}: {dist}m\n"

        objects_text = ""
        for obj_id, info in obs["objects"].items():
            objects_text += f"  [{obj_id}]: position={info['position']}, size={info.get('size')}\n"

        return f"""SCENE REPORT:
{robots_text}
Objects:
{objects_text}
Goal: {obs['goal']}
"""

    def _build_role_assignment_prompt(self, enhanced_obs: str) -> str:
        return f"""You are a multi-robot task coordinator.

{enhanced_obs}

Your task:
1. Decompose the goal into sub-tasks
2. Assign each sub-task to the most suitable robot
3. Generate actions for each robot

Use this output format:
<TASK_DECOMPOSITION>
Sub-task 1: ... → Robot [X] (reason)
Sub-task 2: ... → Robot [Y] (reason)
Dependencies: ...
</TASK_DECOMPOSITION>

<ACTION_PLAN>
Robot [X]:
  Step 1: move_to(...)
  Step 2: grasp(...)
Robot [Y]:
  Step 1: move_to(...)
</ACTION_PLAN>
"""
```

### 创新点4：经验记忆库与Few-shot提示（Experience Memory with Few-Shot）

#### 核心思想
构建一个**经验记忆库**（JSON 文件），存储历史成功案例的 `(状态描述, 动作序列, 成功标志)`。每次调用 LLM 时，自动检索与当前状态最相似的 k 个成功案例，注入 Prompt 作为 few-shot 示例，引导 LLM 产出正确决策。

#### 为什么出彩（评委视角）

1. **Few-shot learning 的工程化应用**：将 ICL（In-Context Learning）与外部记忆结合，是当前 LLM Agent 的研究热点。
2. **"越用越聪明"**：随着运行次数增加，记忆库积累更多案例，系统性能持续提升 —— 这在报告中是很好的故事线。
3. **技术成熟度高**：RAG（检索增强生成）是当前工业界最成熟的 LLM 应用范式，有大量参考实现。

#### 实现难度
⭐⭐（2/5）—— 核心是文件 I/O + 简单的相似度计算，无复杂组件。

#### 成功率提升预期
+10%~20%（随着记忆库增长可持续提升）

```python
# --- memory_manager.py ---
import json
import os
from datetime import datetime

class ExperienceMemory:
    def __init__(self, memory_path="memory.json", embedding_dim=12):
        self.memory_path = memory_path
        self.memory = self._load_memory()

    def _load_memory(self) -> dict:
        if os.path.exists(self.memory_path):
            with open(self.memory_path, "r", encoding="utf-8") as f:
                return json.load(f)
        return {"version": "1.0", "cases": []}

    def _save_memory(self):
        with open(self.memory_path, "w", encoding="utf-8") as f:
            json.dump(self.memory, f, ensure_ascii=False, indent=2)

    def add_case(self, obs: dict, actions: dict, success: bool,
                 completion_time: float = 0):
        state_vec = self._state_to_vector(obs)
        case = {
            "id": f"case_{len(self.memory['cases'])+1:04d}",
            "timestamp": datetime.now().isoformat(),
            "scenario": obs.get("task_type", "unknown"),
            "state_snapshot": self._extract_key_state(obs),
            "state_embedding": state_vec,
            "actions": actions,
            "success": success,
            "completion_time": completion_time,
            "tags": self._generate_tags(obs),
        }
        self.memory["cases"].append(case)
        self._save_memory()

    def retrieve(self, current_obs: dict, k: int = 3) -> list:
        """
        检索 k 个最相似的成功案例
        """
        if not self.memory["cases"]:
            return []

        current_vec = self._state_to_vector(current_obs)
        successes = [c for c in self.memory["cases"] if c["success"]]

        # 计算余弦相似度
        scored = []
        for case in successes:
            sim = self._cosine_similarity(current_vec, case["state_embedding"])
            scored.append((sim, case))

        scored.sort(key=lambda x: x[0], reverse=True)
        return [case for _, case in scored[:k]]

    def _state_to_vector(self, obs: dict) -> list:
        """
        将观测状态展平为向量（用于相似度计算）
        关键：只取位置信息，忽略噪声
        """
        vec = []
        # 任务目标的对象位置
        for obj_id in sorted(obs.get("objects", {}).keys()):
            pos = obs["objects"][obj_id].get("position", [0, 0, 0])
            vec.extend(pos)
        # 机器人位置
        for r_id in sorted(obs.get("robots", {}).keys()):
            pos = obs["robots"][r_id].get("position", [0, 0, 0])
            vec.extend(pos)
        # 补齐到固定长度
        while len(vec) < 12:
            vec.append(0.0)
        return vec[:12]

    def _cosine_similarity(self, a: list, b: list) -> float:
        dot = sum(ai * bi for ai, bi in zip(a, b))
        norm_a = sum(ai**2 for ai in a) ** 0.5
        norm_b = sum(bi**2 for bi in b) ** 0.5
        if norm_a == 0 or norm_b == 0:
            return 0.0
        return dot / (norm_a * norm_b)

    def _extract_key_state(self, obs: dict) -> dict:
        return {
            "task_goal": obs.get("goal", ""),
            "num_robots": len(obs.get("robots", {})),
            "objects_positions": {
                oid: info["position"]
                for oid, info in obs.get("objects", {}).items()
            },
            "robots_positions": {
                rid: info["position"]
                for rid, info in obs.get("robots", {}).items()
            },
        }

    def _generate_tags(self, obs: dict) -> list:
        tags = []
        tags.append(f"{len(obs.get('robots',{}))}_robots")
        tags.append(f"{len(obs.get('objects',{}))}_objects")
        tags.append(obs.get("task_type", "unknown"))
        return tags


# --- plan_prompter.py 中集成 ---
class MemoryAugmentedPlanner:
    def __init__(self, model="qwen2.5:7b", memory_path="memory.json"):
        self.model = model
        self.memory = ExperienceMemory(memory_path)

    def plan(self, obs: dict) -> dict:
        # 检索相似成功案例
        similar_cases = self.memory.retrieve(obs, k=3)

        # 将案例格式化为 few-shot 示例
        few_shot_text = self._format_few_shot_examples(similar_cases)

        # 构建增强 Prompt
        prompt = self._build_augmented_prompt(obs, few_shot_text)

        response = self._call_llm(prompt)
        actions = self._parse_actions(response)

        return actions

    def _format_few_shot_examples(self, cases: list) -> str:
        if not cases:
            return "（无历史案例）"

        text = "=== 历史成功案例（参考，请学习其策略）===\n"
        for i, case in enumerate(cases):
            text += f"""
--- 成功案例 {i+1} ---
场景: {case['scenario']}
状态: {json.dumps(case['state_snapshot'], ensure_ascii=False)}
采取的动作:
{json.dumps(case['actions'], indent=2, ensure_ascii=False)}
成功完成时间: {case['completion_time']}s
---
"""
        text += "\n请参考以上成功案例的策略，但要根据当前实际情况调整。\n"
        return text

    def _build_augmented_prompt(self, obs: dict, few_shot: str) -> str:
        return f"""{few_shot}

=== 当前场景 ===
目标: {obs['goal']}
机器人: {json.dumps(obs['robots'], ensure_ascii=False)}
物体: {json.dumps(obs['objects'], ensure_ascii=False)}

请根据当前场景和参考案例，生成动作计划。
输出 JSON 格式:
{{"robot_1": [{{"type": "...", "target": [...]}}], ...}}
"""

    def execute_and_record(self, obs: dict, env):
        """
        执行动作并自动记录到记忆库
        """
        plan = self.plan(obs)
        result = env.step(plan["actions"])
        success = result.get("task_complete", False)
        completion_time = result.get("time", 0)

        # 自动记录
        self.memory.add_case(
            obs=obs,
            actions=plan["actions"],
            success=success,
            completion_time=completion_time,
        )

        return {"plan": plan, "success": success, "result": result}
```

### 创新点5：碰撞预测与安全路径规划（Collision Prediction via LLM）

#### 核心思想
在每次规划动作时，让 LLM 同时输出"预期碰撞风险报告"，预判可能的碰撞（机器人与机器人、机器人与物体、机器人与环境边界），并根据风险等级主动调整运动路径。

#### 为什么出彩（评委视角）

1. **安全性是具身智能的第一优先级**：真实世界中碰撞意味着设备损坏，评委天然关注安全机制。
2. **LLM 做"风险评估"是其强项之一**：不需要精确物理计算，LLM 擅长基于常识做定性判断。
3. **与 MuJoCo 物理引擎形成互补**：物理引擎只能检测碰撞（事后），LLM 可以预测碰撞（事前）。

#### 实现难度
⭐⭐⭐（3/5）—— 需要维护世界模型、计算碰撞概率，但核心仍是 Prompt。

#### 成功率提升预期
+5%~10%（主要在密集多机器人场景中）

#### 详细实现方案

**架构设计：**

```python
# --- plan_prompter.py ---

class CollisionAwarePlanner:
    def __init__(self, model="qwen2.5:7b", safe_radius=0.15):
        self.model = model
        self.safe_radius = safe_radius  # 机器人+物体安全半径
        self.trajectory_buffer = {}      # 记录各机器人移动轨迹

    def plan(self, obs: dict) -> dict:
        # Step 1: 分析当前碰撞风险
        risk_report = self._generate_risk_report(obs)

        # Step 2: 构建碰撞感知 Prompt
        prompt = self._build_collision_aware_prompt(obs, risk_report)

        # Step 3: LLM 生成安全路径
        response = self._call_llm(prompt)

        # Step 4: 解析并双重验证
        actions, safe_path = self._parse_collision_response(response)

        # Step 5: 代码层二次验证
        verified = self._verify_no_collision(actions, obs)

        # 更新轨迹缓冲
        self._update_trajectory_buffer(actions)

        return {"actions": verified, "risk_report": risk_report}

    def _generate_risk_report(self, obs: dict) -> str:
        """基于距离的简单碰撞风险预判"""
        robots = obs["robots"]
        robot_ids = list(robots.keys())
        risks = []

        # 机器人间碰撞检测
        for i in range(len(robot_ids)):
            for j in range(i+1, len(robot_ids)):
                r1, r2 = robot_ids[i], robot_ids[j]
                dist = self._distance(
                    robots[r1]["position"], robots[r2]["position"]
                )
                if dist < self.safe_radius * 4:
                    risks.append(
                        f"⚠️ robots [{r1}] & [{r2}] 仅相距 {dist:.2f}m - 碰撞风险!"
                    )

        # 机器人与物体碰撞检测
        for robot_id, r_info in robots.items():
            for obj_id, obj_info in obs.get("objects", {}).items():
                dist = self._distance(r_info["position"], obj_info["position"])
                if dist < self.safe_radius * 2:
                    risks.append(
                        f"⚠️ robot [{robot_id}] 接近物体 [{obj_id}] (距离 {dist:.2f}m)"
                    )

        # 边界检测
        for robot_id, r_info in robots.items():
            pos = r_info["position"]
            if abs(pos[0]) > 1.8 or abs(pos[1]) > 1.8:
                risks.append(f"⚠️ robot [{robot_id}] 接近工作空间边界")

        if not risks:
            return "✅ 当前无碰撞风险"

        return "碰撞风险报告:\n" + "\n".join(risks)

    def _build_collision_aware_prompt(self, obs: dict, risk_report: str) -> str:
        return f"""你是机器人安全路径规划器。

{risk_report}

场景信息:
{json.dumps(obs, ensure_ascii=False, indent=2)}

安全规则:
1. 任意两个机器人的路径间距必须 > 0.3m
2. 机器人移动速度上限 0.5 m/s
3. 不得穿越物体（物体半径约 0.1m）
4. 工作空间范围: x∈[-2, 2], y∈[-2, 2]

请按以下格式输出:
<COLLISION_PREDICTION>
列出所有可能的碰撞风险对（机器人-机器人、机器人-物体、机器人-边界）
每个风险标注概率（HIGH/MEDIUM/LOW）和交汇点坐标
</COLLISION_PREDICTION>

<SAFE_ACTIONS>
{{
  "robot_1": [{{"type": "move", "target": [x,y,z], "wait_before": N}}],
  "robot_2": [{{"type": "move", "target": [x,y,z], "wait_before": N}}]
}}
"wait_before" 表示执行前等待的步数（用于避让）
</SAFE_ACTIONS>
"""

    def _verify_no_collision(self, actions: dict, obs: dict) -> dict:
        """
        二次校验：模拟执行动作序列，检测是否有路径交叉
        """
        # 提取所有移动路径
        paths = {}
        for robot_id, action_list in actions.items():
            paths[robot_id] = [obs["robots"][robot_id]["position"]]
            for action in action_list:
                if action["type"] == "move":
                    paths[robot_id].append(action["target"])

        # 检查路径段是否有交叉（在 XY 平面上的近似）
        verified = {}
        for robot_id, action_list in actions.items():
            verified[robot_id] = []
            for i, action in enumerate(action_list):
                if action["type"] == "move":
                    target = action["target"]
                    # 检查目标是否与其他机器人太近
                    for other_id in actions:
                        if other_id != robot_id:
                            other_pos = paths[other_id][-1]
                            if self._distance(target, other_pos) < self.safe_radius * 3:
                                # 注入避让动作
                                escape_point = self._find_escape(target, other_pos)
                                verified[robot_id].append(
                                    {"type": "move", "target": escape_point}
                                )
                    verified[robot_id].append(action)
                else:
                    verified[robot_id].append(action)
        return verified

    def _distance(self, a, b):
        return sum((ai - bi)**2 for ai, bi in zip(a, b)) ** 0.5

    def _find_escape(self, my_target, obstacle_pos):
        """向远离障碍物的方向偏移"""
        dx = my_target[0] - obstacle_pos[0]
        dy = my_target[1] - obstacle_pos[1]
        norm = (dx**2 + dy**2) ** 0.5 or 1.0
        return [
            my_target[0] + dx/norm * self.safe_radius * 3,
            my_target[1] + dy/norm * self.safe_radius * 3,
            my_target[2],
        ]
```

```python
# --- prompt_optimizer.py ---
import json
import os
from datetime import datetime

class PromptOptimizer:
    def __init__(self, model="qwen2.5:7b", prompt_dir="prompts/"):
        self.model = model
        self.prompt_dir = prompt_dir
        self.current_prompt_version = self._load_latest_version()
        self.failure_log = []
        self.success_log = []

    def _load_latest_version(self) -> dict:
        """加载当前最新的 Prompt 版本"""
        os.makedirs(self.prompt_dir, exist_ok=True)
        version_file = os.path.join(self.prompt_dir, "version_manifest.json")

        if os.path.exists(version_file):
            with open(version_file, "r") as f:
                manifest = json.load(f)
            latest = manifest["latest"]
            prompt_file = os.path.join(self.prompt_dir, f"prompt_v{latest:03d}.json")
            with open(prompt_file, "r") as f:
                return json.load(f)
        else:
            # 返回初始版本
            return {
                "version": 1,
                "system_prompt": self._get_initial_prompt(),
                "history": ["Initial prompt"],
                "score": 0.0,
            }

    def _get_initial_prompt(self) -> str:
        return """你是一个多机器人控制系统的决策核心。根据当前观测，
输出每个机器人的下一步动作。"""

    def log_result(self, success: bool, obs: dict, actions: dict, result: dict):
        """记录每次执行结果"""
        entry = {
            "timestamp": datetime.now().isoformat(),
            "success": success,
            "observation_summary": self._summarize_obs(obs),
            "actions_taken": actions,
            "result_summary": self._summarize_result(result),
        }
        if success:
            self.success_log.append(entry)
        else:
            self.failure_log.append(entry)

    def analyze_and_optimize(self, min_samples: int = 10) -> dict:
        """
        当失败案例达到阈值后，触发元分析
        """
        if len(self.failure_log) < min_samples:
            return {"status": "insufficient_data"}

        # Step 1: Meta-Analysis —— LLM 分析失败模式
        analysis = self._meta_analyze_failures()

        # Step 2: Prompt Optimization —— LLM 生成优化建议
        optimization = self._generate_prompt_optimization(analysis)

        # Step 3: 生成新版 Prompt
        new_prompt = self._apply_optimization(optimization)

        # Step 4: 保存新版本
        new_version = self._save_new_version(new_prompt, optimization)

        return {
            "status": "optimized",
            "old_version": self.current_prompt_version["version"],
            "new_version": new_version,
            "analysis": analysis,
            "changes": optimization.get("changes_summary", ""),
        }

    def _meta_analyze_failures(self) -> dict:
        """元分析：让 LLM 分析所有失败案例，找出共性模式"""
        # 取最近 20 个失败案例
        recent_failures = self.failure_log[-20:]
        failures_text = json.dumps(recent_failures, ensure_ascii=False, indent=2)

        meta_prompt = f"""你是 Prompt Engineering 专家。分析以下机器人控制失败案例，
找出当前 Prompt 的设计缺陷。

当前 Prompt:
```
{self.current_prompt_version['system_prompt']}
```

失败案例（最近 20 个）:
{failures_text}

请分析：
1. 失败模式分类（如：空间推理错误、动作顺序错误、忽略障碍物等）
2. 每种模式的频率
3. 当前 Prompt 中缺失的关键引导信息
4. 失败是否与 Prompt 的表述模糊有关

输出 JSON:
{{
  "failure_patterns": [
    {{"pattern": "...", "frequency": N, "examples": ["..."]}}
  ],
  "prompt_deficiencies": [
    "缺失XXX约束",
    "未强调XXX优先级"
  ],
  "root_analysis": "..."
}}
"""
        resp = self._call_llm(meta_prompt)
        return self._extract_json(resp)

    def _generate_prompt_optimization(self, analysis: dict) -> dict:
        """基于元分析结果，生成 Prompt 优化方案"""
        opt_prompt = f"""你是 Prompt Engineering 专家。基于以下失败分析，优化机器人控制 Prompt。

失败分析:
{json.dumps(analysis, ensure_ascii=False, indent=2)}

当前 Prompt:
```
{self.current_prompt_version['system_prompt']}
```

历史优化记录:
{self.current_prompt_version.get('history', [])}

设计新 Prompt 时注意:
1. 针对高频失败模式添加明确的约束/引导
2. 使用具体的示例说明期望行为
3. 保持 Prompt 简洁（不超过 2000 tokens）
4. 每次只优化 1-2 个关键点，避免过度修改

输出 JSON:
{{
  "new_system_prompt": "优化后的完整 prompt 文本",
  "changes_summary": "本次修改了哪些内容，为什么",
  "expected_improvement": "预期如何解决失败问题"
}}
"""
        resp = self._call_llm(opt_prompt)
        return self._extract_json(resp)

    def _apply_optimization(self, optimization: dict) -> dict:
        """应用优化，生成新的 Prompt 版本"""
        new_version = {
            "version": self.current_prompt_version["version"] + 1,
            "system_prompt": optimization["new_system_prompt"],
            "history": self.current_prompt_version.get("history", []) + [
                f"v{self.current_prompt_version['version']}→v{self.current_prompt_version['version']+1}: {optimization['changes_summary']}"
            ],
            "score": 0.0,
            "based_on_analysis": optimization.get("expected_improvement", ""),
        }
        return new_version

    def _save_new_version(self, new_prompt: dict, optimization: dict) -> int:
        """保存新版本到磁盘"""
        version = new_prompt["version"]
        prompt_file = os.path.join(self.prompt_dir, f"prompt_v{version:03d}.json")
        with open(prompt_file, "w", encoding="utf-8") as f:
            json.dump(new_prompt, f, ensure_ascii=False, indent=2)

        # 更新清单
        manifest_file = os.path.join(self.prompt_dir, "version_manifest.json")
        manifest = {"latest": version, "versions": list(range(1, version + 1))}
        with open(manifest_file, "w") as f:
            json.dump(manifest, f)
        return version

    def a_b_test(self, old_prompt: dict, new_prompt: dict,
                 test_cases: list, n_runs: int = 10) -> dict:
        """
        A/B 测试：在相同场景下对比新旧 Prompt
        """
        results = {"old": [], "new": []}
        for _ in range(n_runs):
            for case in test_cases:
                # 用旧 Prompt 执行
                old_result = self._run_with_prompt(case, old_prompt["system_prompt"])
                results["old"].append(old_result)

                # 用新 Prompt 执行
                new_result = self._run_with_prompt(case, new_prompt["system_prompt"])
                results["new"].append(new_result)

        # 统计
        old_success = sum(1 for r in results["old"] if r["success"])
        new_success = sum(1 for r in results["new"] if r["success"])

        return {
            "old_prompt_success_rate": old_success / len(results["old"]),
            "new_prompt_success_rate": new_success / len(results["new"]),
            "improvement": (new_success - old_success) / len(results["old"]),
            "verdict": "ACCEPT" if new_success > old_success else "REJECT",
        }

    def _summarize_obs(self, obs: dict) -> str:
        return f"Goal: {obs.get('goal')}, Robots: {len(obs.get('robots',{}))}, Objects: {len(obs.get('objects',{}))}"

    def _summarize_result(self, result: dict) -> str:
        return json.dumps(result, ensure_ascii=False)

    def _call_llm(self, prompt: str) -> str:
        import ollama
        resp = ollama.generate(model=self.model, prompt=prompt)
        return resp["response"]

    def _extract_json(self, text: str) -> dict:
        import re
        match = re.search(r'\{.*\}', text, re.DOTALL)
        if match:
            try:
                return json.loads(match.group(0))
            except:
                pass
        return {}
```

## 四、推荐组合方案

### 最佳组合：创新点2（结构化CoT）+ 创新点4（经验记忆库）

**为什么这是最出彩且最可行的组合？**

| 维度 | 创新点2（CoT）| 创新点4（Memory）| 组合效果 |
|------|------------|---------------|---------|
| 实现难度 | ⭐⭐ | ⭐⭐ | 总计 ⭐⭐（极低门槛）|
| 成功率提升 | +10~20% | +10~20% | **+25~40%（叠加效应）** |
| 可解释性 | 极高（四阶段可追踪） | 极高（可回溯参考案例） | **报告中展示空间大** |
| 技术深度 | CoT 推理链 | RAG 思想 | **覆盖当前 LLM 两大主流范式** |
| 后续扩展 | 可与任何创新点叠加 | 自然增长（越用越好） | **可持续优化** |

**协同原理：**

1. **Memory 提供"经验锚点"** —— 检索到的成功案例为 CoT 推理提供了正确方向的指引，减少了 CoT 方向跑偏的概率。
2. **CoT 提供"推理模板"** —— 结构化四阶段输出确保即使参考案例不完美，LLM 也不会直接复制错误动作，而是通过推理调整。
3. **互补性** —— Memory 解决"之前遇到过类似情况没"的问题（经验驱动），CoT 解决"面对新情况如何推理"的问题（逻辑驱动）。

```python
class CoTMemoryPlanner:
    def __init__(self):
        self.memory = ExperienceMemory("memory.json")  # 创新点4
        self.cot_format = SPATIAL_COT_PROMPT           # 创新点2

    def plan(self, obs):
        # 1. 检索相似案例（创新点4）
        similar_cases = self.memory.retrieve(obs, k=3)
        few_shot = self._format_examples(similar_cases)

        # 2. 构建 CoT + Few-shot Prompt（创新点2 + 4）
        prompt = f"""
{few_shot}

=== 当前场景 ===
{obs}

{cot_format}
"""
        response = self._call_llm(prompt)

        # 3. 解析 CoT 结构
        sections = self._parse_cot(response)

        # 4. 记录到记忆库
        self.memory.add_case(obs, sections["ACT"], success, time)

        return sections["ACT"]
```

| 组合 | 问题 |
